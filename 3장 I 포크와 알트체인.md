# 3장 I 포크와 알트체인


### 1. 비트코인 개선 과정: BIP (Bitcoin Improvement Proposals)
비트코인은 오픈소스 프로젝트로서 특정 소유자나 결정권자가 없으므로, 커뮤니티 전체의 합의를 거치는 표준화된 개선 절차인 BIP를 따른다.

* **BIP의 수명주기:**
    1.  **제안:** 누구나 아이디어를 내고 BIP 편집자에게 제출한다.
    2.  **초안(Draft):** 편집자가 승인하면 공식 초안 상태가 된다.
    3.  **지시(Signaling):** 채굴자들이 해당 안건에 대해 지지 신호를 보낸다.
    4.  **최종(Final):** 채굴자의 **95%(최근 2,016개 블록 기준)**가 동의하면 소프트웨어가 업데이트되고 채택된다.
* **민주적 특성:** 채굴자는 해시레이트(연산 능력)에 비례하여 투표권을 가지며, 압도적인 다수의 동의가 있어야만 프로토콜이 변경된다.


### 2. 포크(Fork)의 개념과 종류
포크는 기존 소프트웨어 코드를 복제하여 새로운 방향으로 수정하는 것을 의미한다.

* **소프트웨어 포크:** 단순히 코드를 가져와 매개변수를 바꾸는 일반적인 방식이다(예: 리눅스 배포판들).
* **소프트포크(Soft Fork):** 이전 버전과 호환성이 유지되는 업데이트다. 모든 채굴자가 즉시 참여할 필요는 없다.
* **하드포크(Hard Fork):** 이전 버전과 호환되지 않는 중대한 변경이다. 모든 채굴자가 새로운 소프트웨어로 업그레이드해야 한다.
* **논쟁의 여지가 있는 하드포크:** 커뮤니티 내 의견 합의가 이루어지지 않은 상태에서 강행되는 하드포크다. 이 경우 네트워크가 두 개의 독립된 블록체인으로 영구적으로 갈라지게 된다.


### 3. 주요 하드포크 사례 및 리플레이 공격
**비트코인캐시(BCH)와 비트코인SV(BSV)**
* **배경:** 비트코인의 1MB 블록 크기 제한으로 인한 처리 속도 저하와 수수료 상승 문제를 해결하려는 시도에서 시작되었다.
* **분화:**
    * **비트코인(BTC):** 블록 크기를 유지하되 효율성을 개선(세그윗 등)하자는 쪽이다.
    * **비트코인캐시(BCH):** 블록 크기를 직접 늘려(8MB→32MB) 저렴하고 빠른 거래를 지향하는 쪽이다.
    * **비트코인SV(BSV):** BCH에서 다시 분화되어 더 큰 블록(128MB)과 사토시 나카모토의 원래 비전을 강조한다.
* **특징:** 포크 시점에 비트코인을 보유했던 사람은 동일한 양의 새로운 코드를 받게 된다.

**리플레이 공격(Replay Attack)과 방어**
* **현상:** 체인이 갈라졌을 때 한쪽 체인에서의 거래 데이터를 복사해 다른 체인에 전송하면, 의도치 않게 양쪽 체인에서 코드가 전송되는 현상이다.
* **해결책:** 비트코인캐시는 **SIGHASH_FORKID**라는 필드를 추가하여 한 체인의 서명이 다른 체인에서는 유효하지 않도록 설계해 이를 방지했다.



### 4. 알트코인(Altcoin)의 역사와 실험
비트코인 코드를 기반으로 하거나 새로운 아이디어를 결합한 다양한 대체 코인들이 등장했다.

* **초기 실험적 코인:**
    * **아이엑스코인(Ixcoin):** 개발자가 미리 채굴(프리마인)한 사례다.
    * **솔리드코인(Solidcoin):** 빠른 블록 생성과 고정 수수료를 시도했으나 스팸 공격에 취약해 실패했다.
    * **제이스트겔드(Geistgeld):** 15초의 짧은 블록 타임으로 인해 미확인 블록(고아 블록)이 대량 발생했다.
* **주요 알트코인:**
    * **네임코인(Namecoin):** 탈중앙화 DNS(도메인 네임 시스템) 구축을 시도했다.
    * **프라임코인(Primecoin):** 작업 증명 과정에서 소수(Prime number)를 찾는 유용한 연산을 수행하도록 설계되었다.
    * **라이트코인(Litecoin):** "비트코인이 금이라면 라이트코인은 은"이라는 개념으로 등장했다. 4배 빠른 블록 생성과 스크립트(Scrypt) 알고리듬으로 ASIC 저항성을 시도했다.
    * **도지코인(Dogecoin):** 인터넷 밈에서 시작된 코인으로 공급량 제한이 없다.



### 5. 차세대 체인과 특화 기능
**'2.0' 체인 및 플랫폼**
단순한 화폐를 넘어 플랫폼 역할을 지향한다.
* **넥스트(NXT):** 컬러 코인(자산에 태그를 붙임) 개념을 도입했다.
* **카운터파티(Counterparty):** 비트코인 블록체인 위에서 스마트 콘트랙트 기능을 구현하려 시도했다.

**개인정보 보호(Privacy) 코인**
* **대시(Dash):** 거래를 혼합하는 '프라이빗샌드' 기능을 제공한다.
* **모네로(Monero):** 링 시그니처와 스텔스 주소로 송수신자와 금액을 은닉한다.
* **지캐시(Zcash):** 영 지식 증명(zk-SNARKs)을 사용하여 데이터 공개 없이 거래 사실만 입증한다.

**비-작업증명(Non-PoW) 코인**
채굴의 중앙화와 에너지 낭비를 해결하고자 한다.
* **리플(Ripple):** 은행 간 국제 송금에 특화되어 있으며, BFT 방식의 합의 알고리듬을 사용한다.
* **스텔라(Stellar):** 리플에서 분사되었다. 비영리 목적의 저비용 결제 네트워크를 지향하며 SCP 합의 메커니즘을 사용한다.



### 6. 블록체인 확장성 솔루션
네트워크 처리 속도를 높이기 위한 기술적 시도들이다.

* **세그윗(SegWit):** 트랜잭션 데이터에서 서명(Witness) 부분을 분리하여 블록 공간을 확보하고, '트랜잭션 가변성' 문제를 해결한 소프트포크다.

* **라이트닝 네트워크:** 블록체인 밖(오프체인)에 양방향 채널을 열어 소액 결제를 실시간으로 처리하고 결과만 기록하는 방식이다.

* **사이드체인/롤업:** 별도의 체인에서 거래를 처리한 후 요약본을 메인 체인에 올리는 방식이다(ZK 롤업, 옵티미스틱 롤업 등).
* **샤딩(Sharding):** 네트워크를 여러 조각(Shard)으로 나누어 트랜잭션을 병렬 처리하는 기술이다.



### 7. 이더리움 클래식(ETC) 사례
2016년 DAO 해킹 사건에 대한 대응으로 발생한 포크다.

* **이더리움(ETH):** 해킹 피해를 복구하기 위해 거래 기록을 되돌리는 하드포크를 선택했다.
* **이더리움 클래식(ETC):** "코드가 곧 법(Code is Law)"이라는 원칙 아래, 어떤 이유로도 블록체인 기록은 수정될 수 없다고 주장하며 기존 체인을 유지했다.
* 이 과정에서도 리플레이 보호가 즉각 적용되지 않아 사용자가 스스로 자금을 분산시켜야 했던 혼란이 있었다.



### 요약
비트코인은 BIP를 통해 엄격한 합의 과정을 거쳐 발전하며, 이 과정에서 의견 차이가 발생하면 하드포크를 통해 새로운 암호화폐(BCH 등)가 탄생한다. 또한, 기술적 한계를 극복하기 위해 라이트코인, 대시, 리플 같은 다양한 알트코인과 세그윗, 라이트닝 네트워크 같은 확장성 기술이 지속적으로 연구되고 있다.
